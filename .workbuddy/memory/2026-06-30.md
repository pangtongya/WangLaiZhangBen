## 修复 WangLaiZhangBen iOS 项目构建问题

项目初始状态：BUILD FAILED（多个编译错误），修复后 BUILD SUCCEEDED（零警告）

### 修复的问题清单：

1. **formatAmount 访问控制** — SummaryCardView.swift 中声明为 `private func`，但被 BillListView/CompareView 引用，改为 internal

2. **CategoryInfo 不符合 Identifiable** — ForEach(CATEGORIES) 需要 Identifiable，添加了 `var id: String { key }`

3. **PersonStats 不符合 Identifiable** — ForEach(stats) 需要 Identifiable，添加了 `var id: String { person.id }`

4. **presentationMode 废弃** — AddBillView/PersonEditView 使用了废弃的 `@Environment(\.presentationMode)`，替换为 `@Environment(\.dismiss)`

5. **onChange API 废弃** — onChange(of:perform:) 在 iOS 17 废弃，改为 `onChange(of:) { _, newValue in }`

6. **SectorMark iOS 17 要求** — SectorMark 需要 iOS 17+，将 deployment target 从 16.0 升到 17.0

7. **LazyVGrid API 错误** — PersonEditView 中 `Array(repeating: GridItem(.fixed(32)), spacing: 8)` 参数错误，应为 `Array(repeating:count:)`，同时修复为 flexible grid

8. **Charts 框架引用** — pbxproj 引用了不存在的本地 Charts.framework，替换为系统 SDK 框架 (SDKROOT)
   - 删除了本地 Charts 文件引用、Embed Frameworks 阶段
   - 添加了系统 Charts.framework 链接
   - 清理了 FRAMEWORK_SEARCH_PATHS

9. **pbxproj 重复组引用** — WangLaiZhangBen 源码组被引用了两次（root group 和 wrapper group），删除 root group 中的直接引用

10. **CompareView PBXBuildFile 误删** — 在清理 Charts 引用时误删了 CompareView.swift 的 Sources 条目，已恢复

## 功能层彻底整改

用户反馈功能全部不可用，经全面审查发现 7 个核心功能 Bug：

### 修复清单（第二轮）：

11. **currentPid 不持久化（致命）** — AccountViewModel.loadData() 从未从 UserDefaults 恢复 currentPid，每次启动都重置为第一个人物，人物切换完全无效。修复：loadData() 中先读取 dataManager.currentPid()

12. **BillRecord ID 碰撞** — 使用 Int(timestamp*1000) 作 ID，同一毫秒多条记录会碰撞。修复：DataManager.addBill() 检查现有 ID 并递增

13. **CompareView 标签语义颠倒** — "take" 字段实际是"我给TA"但标签写"我从TA那收到的"。修复：标签改为"我给TA的钱"，同时修正 net 值符号逻辑

14. **UIKit Alert 替换为 SwiftUI** — clearCurrentPersonBills 使用 UIAlertController，在 SwiftUI 中不可靠。替换为 .alert modifier

15. **静默失败无反馈** — AddBillView 金额无效/未选分类时静默失败；MeView 导入无提示；PersonEditView 空名称无提示。全部添加了 alert 反馈

16. **新建人物后不切换** — addPerson 只在 currentPid 为空时才切换。修复：始终切换到新建人物

17. **ViewModel 数据同步重构** — 每次 CRUD 都从 UserDefaults 重新加载整个数组，效率低且易不一致。改为：本地数组直接 mutate + 写到 UserDefaults（ViewModel 是数据权威来源）
